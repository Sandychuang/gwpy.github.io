FFT routines for GWpyÂ¶

The Fourier transform (and the associated digital FFT algorithm) is the most common way of investigating the frequency-domain content of a time-domain signal. GWpy provides wrappers of average spectrum methods from scipy.signal, and lal to simplify calculating a FrequencySeries from a TimeSeries.

The gwpy.signal.fft sub-package provides the following PSD averaging methods:

gwpy.signal.fft.scipy.welch Calculate a PSD of this TimeSeries using Welch’s method.
gwpy.signal.fft.scipy.bartlett Calculate a PSD of this TimeSeries using Bartlett’s method
gwpy.signal.fft.lal.welch Calculate an PSD of this TimeSeries using Welch’s method
gwpy.signal.fft.lal.bartlett Calculate an PSD of this TimeSeries using Bartlett’s method
gwpy.signal.fft.lal.median Calculate a PSD of this TimeSeries using a median average method
gwpy.signal.fft.lal.median_mean Calculate a PSD of this TimeSeries using a median-mean average method

Each of these can be specified by passing the function name as the method keyword argument to any of the relevant TimeSeries instance methods, e.g:

>>> ts = TimeSeries(...)
>>> psd = ts.psd(..., method='welch', ...)

Note

Since welch and bartlett are defined in both the scipy and lal sub-modules, the LAL versions are registered as 'lal-welch' and 'lal-bartlett', so to use them, pass method='lal-welch' to the relevant TimeSeries method.
